# 🛡️ Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

If you discover a potential security vulnerability in this project, please report it immediately by sending an email to company@dezinet.ltd. 

**Please do not open a public issue for security-related reports.**

Once we receive your report, our security team will:

1. Confirm receipt of the report within 24–48 hours.
2. Investigate the vulnerability and determine the severity.
3. Develop a fix/mitigation.
4. Notify you of our findings and the planned release schedule.
5. Provide public credit for your responsible disclosure in our [Changelog](file:///doc/CHANGELOG.md).

## Disclosing Responsibility

We appreciate the efforts of security researchers and ask that you:

* Do not perform testing that impact users beyond yourself.
* Give us a reasonable amount of time to resolve the issue before any public disclosure.
* Provide detailed steps to reproduce the vulnerability.
